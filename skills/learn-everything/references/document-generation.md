# 文档生成

## 概念选择

从 knowledge-graph.json 中按优先级选择下一个概念（见 knowledge-graph-spec.md 中的优先级规则）。

选中后：
- 将该节点 status 更新为 learning
- 更新 current-session.json：currentConcept → 选定概念

## 前置检查

检查选定概念的 requires 边：
- 所有前置概念 confidence ≥ 0.5 → 继续
- 有前置概念 confidence < 0.5 → 切换到该前置概念优先学习

## 文档结构

为选定概念生成 500-1000 字的聚焦学习文档。保存到 `docs/{concept-id}.md`。

### 固定结构

```markdown
# {概念名称}

## 定义
用一句话精确定义这个概念。

## 为什么重要
这个概念在整个知识体系中的位置和作用。为什么需要理解它。

## 核心原理
概念的核心内容。这是文档的主体部分。
用清晰的逻辑展开，必要时分小节。

## 与其他概念的关联
这个概念如何连接到你已经学过或即将学到的其他概念。
明确标注 requires / relates-to / extends 关系。

## 边界与常见误解
这个概念不是什么。容易和什么搞混。常见的错误理解。
```

### 长度控制

- 500 字下限：确保有足够的深度
- 1000 字上限：保持聚焦，不发散
- 如果概念太大，拆分为多个子概念分别生成文档

## 风格适配

根据 learner-profile.json 调整文档风格：

| 画像条件 | 文档调整 |
|---------|---------|
| trait: 偏好类比 | 每个核心概念配一个贴切的类比 |
| trait: 偏好从细节到整体 | 先讲具体例子，再总结抽象原理 |
| scaffoldingLevel: high | 增加过渡句、分步解释、更多示例 |
| scaffoldingLevel: low | 直接切入核心，减少铺垫 |
| 有相关 weakness 记录 | 在"边界与常见误解"中针对性展开 |

## LLM 自行生成

用户提供的是概念名称或领域名词，不是原始材料。LLM 基于自身知识直接生成学习文档。

如果用户在项目中创建了 `raw/` 目录并放入参考材料，文档生成时应参考这些材料，但这不是必需的。

## 生成后

1. 文档保存到 `docs/{concept-id}.md`
2. 将文档内容展示给用户阅读
3. 等用户表示读完后进入下一阶段
4. 更新 current-session.json：stage → "verification"
