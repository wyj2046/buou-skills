# 诊断引擎

## 冷启动诊断（新领域）

当 knowledge-graph.json 中 nodes 为空时执行。

### 目标

用 5-8 个渐进式问题探测用户起点，确定：
1. 用户对该领域的已有认知水平
2. 用户的思维方式和学习偏好（初始化 learner-profile）
3. 应该从哪个概念开始教学

### 问题设计原则

**广→窄、浅→深：**
1. 先用 1-2 个开放性问题探测整体认知广度
2. 再用 2-3 个针对性问题定位核心概念掌握度
3. 最后用 1-2 个边界问题测试知识深度

**不问"你知道 X 吗"。** 设计能暴露认知结构的问题。例如：
- "你知道维特根斯坦吗？"
- "你觉得'独角兽存在'这句话有没有意义？为什么？"

**一次一个问题。** 不要一次性抛出多个问题。等用户回答后再提下一个。

### 执行流程

1. 基于领域名称，设计第一个广域探测问题
2. 根据用户回答，动态调整后续问题方向
3. 每个回答后，内部评估用户在以下维度的水平：
   - 领域术语熟悉度（0-1）
   - 核心概念理解度（0-1）
   - 系统性思维程度（0-1）
4. 5-8 个问题后，综合评估确定起始点

### 诊断完成后

1. 在 knowledge-graph.json 中创建初始概念节点（5-10 个），标记为 unknown
2. 根据诊断结果选择第一个要学的概念（遵循概念选择优先级）
3. 初始化 learner-profile.json 中的初始 traits
4. 更新 current-session.json：stage → "document-generation"，currentConcept → 选定概念

## 复习诊断（已有领域）

当 knowledge-graph.json 中已有节点时执行。

### 执行流程

1. 执行 confidence 衰减（见 knowledge-graph-spec.md）
2. 找出所有 weak 节点和 confidence < 0.5 的节点
3. 用 2-3 个快速问题确认用户是否还记得之前学的内容
4. 根据回答更新对应节点的 confidence
5. 选择下一个要学的概念（遵循概念选择优先级）
6. 更新 current-session.json

### 复习问题设计

- 针对 weak 或低 confidence 的具体概念提问
- 问题应该能区分"真的忘了"和"还记得但需要巩固"
- 2-3 个问题即可，不要做完整测试
