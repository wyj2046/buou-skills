# 深度追问

## 4 种追问策略

### 1. 追问链（pass 时使用）

在用户正确回答后，沿以下方向追问 2-3 层：

- **层 1 — 为什么：** 追问用户理解的因果关系。"你说 X 是因为 Y，为什么 Y 会导致 X？"
- **层 2 — 如果不：** 反事实思考。"如果 Y 不成立，X 会怎样？"
- **层 3 — 极端情况：** 推到边界。"那如果把 Y 推到极端，X 还成立吗？"

每层之后等用户回答，根据回答质量决定是否继续下一层。

### 2. 反例挑战（pass 时使用，与追问链配合）

提供一个看似矛盾的反例，要求用户调和：

"有人说 Z（看起来与 X 矛盾的观点），你怎么看？"

目的不是推翻用户的理解，而是测试其对边界的把握。

### 3. 缺口定位（partial 时使用）

只追问 1 层，聚焦于用户回答中缺失的那个维度：

"你提到了 A 和 B，但似乎没有涉及 C。你觉得 C 在这里起什么作用？"

不要展开更多追问。1 层定位缺口后进入讨论窗口。

### 4. 边界探测（任何判定后可选使用）

将当前概念与知识图谱中相邻概念连接：

"X 和 Y 之间是什么关系？它们有什么相似和不同？"

用于发现概念间的关联，同时检测是否存在混淆。

## 策略组合逻辑

| 判定 | 使用的策略 | 最大追问层数 |
|------|----------|------------|
| pass | 追问链 + 反例挑战 + 边界探测（可选） | 3 层 |
| partial | 缺口定位 | 1 层 |
| fail | 无追问，直接引导重讲 | 0 层 |

### challengeAppetite 的影响

- high → pass 时追问链用满 3 层，反例挑战必用
- medium → pass 时追问 2 层，反例挑战视情况
- low → pass 时追问 1 层，不用反例挑战

## 追问后

所有追问完成后，**必须** 进入讨论窗口（读 intelligent-feedback.md）。
